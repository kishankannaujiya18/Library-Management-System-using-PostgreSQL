# Library-Management-System-using-PostgreSQL
#  Library Management System | PostgreSQL SQL Project

##  Project Overview

The **Library Management System** is a comprehensive SQL project built using **PostgreSQL**, designed to simulate the operations of a real-world library.

This project demonstrates my understanding of relational database design, SQL programming, and business data analysis by implementing everything from basic database creation to advanced analytical SQL queries.

The objective was not only to manage library records but also to solve practical business problems using SQL.

---

#  Objectives

* Design a normalized relational database
* Build relationships using Primary and Foreign Keys
* Perform complete CRUD operations
* Manage books, members, employees, and branches
* Analyze operational data using SQL
* Practice advanced PostgreSQL concepts
* Solve real-world business scenarios

---

#  Tech Stack

* PostgreSQL

#  Database Structure

The project includes multiple relational tables:

* Branch
* Employees
* Members
* Books
* Issued Status
* Return Status

Each table is connected using appropriate relationships to maintain data integrity.

# Business Problems Solved

This project answers several real-world business questions, including:

* Which books are currently available?
* Which books are issued most frequently?
* Which members have overdue books?
* Which branch issues the highest number of books?
* Which employees manage the most transactions?
* What is the branch-wise rental revenue?
* Which book categories are most popular?
* Which members borrow books most frequently?
* Monthly issue and return trends
* Overall library performance analysis
